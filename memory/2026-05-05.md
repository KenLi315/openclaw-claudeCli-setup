# 2026-05-05 Topaz Wiki 項目討論記錄

## 項目願景：Topaz Wiki
打造一個以「原件完整性」為核心、AI 驅動的動態索引知識庫，拒絕傳統 RAG 的碎片化（Atomic Chunking）邏輯。

## 核心需求
1. **多模態輸入**：支援 txt, pdf, word, url, 圖片, YouTube 連結（逐字稿）。
2. **完整性儲存**：文檔不搣碎、不切片，保留原始脈絡與靈魂，轉化為帶有標籤的 Markdown 原件。
3. **AI 自動編目與索引**：
   - 模仿人類閱讀習慣：建立「書名 -> 目錄 (TOC) -> 索引 (Index)」的層次結構。
   - AI 深度閱讀後自動生成精細目錄與關鍵事實標籤。
4. **Wiki 式反向連結**：跨文檔關聯，自動形成知識拓撲，實現「終極記憶」。
5. **事實搜尋引擎 API**：供其他模型調用，精準定位原文段落並引用，消除幻覺。

## 技術達成共識
- **架構方向**：鎖定 **Vectorless RAG / PageIndex** 邏輯。
- **檢索機制**：兩步跳轉（查目錄定位 -> 加載大塊完整上下文）。
- **補強建議**：結合 **GraphRAG (Neo4j)** 處理 Wiki 關聯，利用 **Long-Context 模型 (Gemini 1.5 Pro)** 直接閱讀定位後的原文。
