| **資源消耗** | 較高（每個容器都有輕量作業系統層）。 | 非常低（僅管理進程，幾乎不佔額外資源）。 |
| **隔離性** | **完全隔離**。服務 A 的套件版本不會影響服務 B。 | **無隔離**。所有服務共享伺服器的 Python 環境（除非你手動指定不同 venv）。 |

### 該如何選擇？

#### 1. 選擇 PM2 的情境：
*   **追求速度**：你想在 5 分鐘內把現在那堆 CMD 窗口收起來。
*   **環境簡單**：你的所有 Python 服務都跑在同一個虛擬環境（venv），或者套件依賴不複雜。
*   **資源有限**：伺服器記憶體非常吃緊，不想跑 Docker。
*   **指令範例**：`pm2 start main.py --interpreter python3 --name "MyService"`

#### 2. 選擇 Docker Compose 的情境：
*   **長期維護**：你預計服務會越來越多，且未來可能搬遷伺服器。
*   **套件衝突**：服務 A 需要 Python 3.8，服務 B 需要 Python 3.11。
*   **完整架構**：你的服務除了 Python，還包含資料庫（如 Redis, PostgreSQL），Docker Compose 可以一鍵啟動整個「生態系」。
*   **自動化部署**：你想實現「程式碼推送到 GitHub 後自動更新伺服器服務」。

assistant: <final>Ken，PM2 的使用邏輯非常直覺。雖然它最初是用 Node.js 開發的，但在 Windows 或 Linux 上管理 Python 服務非常方便。

以下是 PM2 的使用全攻略：

### 1. 安裝 PM2
首先，你的伺服器需要安裝 **Node.js**。安裝完 Node.js 後，開啟終端機執行：
```bash
npm install -g pm2
```

### 2. 啟動 Python 服務
假設你有一個腳本叫 `bot.py`，你不再需要開 CMD 窗口，直接輸入：
```bash
pm2 start bot.py --name "my-service" --interpreter python3
```
*   `--name`: 給這個服務起個名字，方便識別。
*   `--interpreter`: 告訴 PM2 使用 Python 運行（如果是 Windows 且環境變數已設好，直接寫 `python` 即可）。

### 3. 常用管理指令（這就是它取代 CMD 窗口的強大之處）

*   **查看所有服務狀態**：
    ```bash
    pm2 list
    ```
    *(你會看到一個表格，顯示所有服務的名稱、ID、狀態、CPU 和記憶體佔用)*

*   **查看即時日誌 (Log)**：
    *(不用再對著 CMD 窗口看輸出，這能看到所有或指定服務的 print 內容)*
    ```bash
    pm2 logs                # 查看所有服務日誌
    pm2 logs "my-service"   # 只看特定服務
    ```

*   **停止/重啟/刪除服務**：
    ```bash
    pm2 stop my-service
    pm2 restart my-service
    pm2 delete my-service
    ```

*   **查看詳細監控界面**（非常酷炫的儀表板）：
    ```bash
    pm2 monit
    ```

### 4. 進階：一次啟動多個服務 (Ecosystem File)
如果你有 10 個服務，一個個輸入指令太慢了。PM2 支援使用一個配置檔案。

1.  產生配置檔：`pm2 init`
2.  編輯產生的 `ecosystem.config.js`：
```javascript
module.exports = {
  apps : [{
    name: "Trading-Bot",
    script: "./trading.py",
    interpreter: "python3"
  }, {
    name: "Data-Collector",
    script: "./collector.py",
    interpreter: "python3"
  }]
}
```
3.  **一鍵啟動所有**：`pm2 start ecosystem.config.js`

### 5. 伺服器重啟自動啟動
這是最核心的功能，確保伺服器當機重啟後，你的服務會自動跑起來：
1.  手動啟動好所有服務。
2.  執行 `pm2 save` (這會把目前的清單存下來)。
3.  執行 `pm2 startup` 並照著螢幕提示操作即可。

### Chris 的小提醒：
如果你使用的是 Python 的 **虛擬環境 (venv)**，啟動指令的 `--interpreter` 要指向該 venv 路徑，例如：
`pm2 start script.py --interpreter ./venv/bin/python`
